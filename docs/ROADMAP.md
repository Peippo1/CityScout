# CityScout Roadmap

## Done / Current State
- City-first onboarding and destination-scoped experience
- Lessons, phrasebook, pronunciation, and translate
- Explore with categories and top picks
- Map with saved places, category styling, itinerary focus mode, and route visualisation
- Search across POIs and saved places
- AI itinerary generation with save item, save all, persisted itineraries, and regenerate
- Itinerary-to-POI matching with mapped vs unmatched distinction
- SwiftUI + SwiftData app architecture
- FastAPI + OpenAI backend with validation and tests
- Light brand system and branded UI pass

## Near-Term Priorities (Next Release)
- Real-device readiness (backend environment config, stability, networking hardening)
- TestFlight preparation and packaging
- Map centering and device-specific polish
- Itinerary refinement UX (clarity, editing, mapped vs unmatched flow)
- Smarter place resolution for itinerary items
- Account and sync planning for later delivery

## V2 (Focused and Realistic)
- Place resolution improvements at scale (disambiguation, confidence handling)
- Itinerary personalization (pace, interests, time windows)
- Offline-friendly caching for core destination data
- Lightweight account + sync for saved places and itineraries

## V3 (Longer Term)
- Multi-city trip planning
- Booking and availability integrations
- Conversational assistant for in-trip planning and discovery

## Long-Term Direction
CityScout stays city-first and local-first, with AI as an enhancement layer rather than a dependency.
